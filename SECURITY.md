# 安全政策

TGDB 會操作系統層級設定與服務，包含但不限於：

- SSH
- nftables
- Fail2ban
- Nginx / 憑證
- Podman / Quadlet
- 備份與還原
- 定時任務

若你發現安全性問題，請不要直接公開完整細節。

## 回報方式

建議優先使用以下方式私下回報：

1. GitHub Security Advisories（若本倉庫已啟用）
2. 維護者在 GitHub 上提供的私人聯絡方式

請盡量提供：

- 受影響版本或 commit
- 問題描述
- 重現步驟
- 可能影響範圍
- 是否需要特定環境或設定

若暫時沒有私下通道，請只開一個不揭露利用細節的 Issue，並註明需要安全聯絡方式。

## 支援版本

目前以 `main` 分支最新版本為主要維護對象。

| 版本 | 支援狀態 |
|------|----------|
| `main` 最新提交 | 支援中 |
| 舊版 / 未同步版本 | 最佳努力，不保證修補 |

## 安全回報範圍

以下問題特別歡迎回報：

- 權限提升
- 認證繞過
- 預設暴露高風險服務
- 憑證、密碼、Token 洩漏
- 備份或還原造成敏感資料外洩
- 防火牆、SSH、Fail2ban 規則失效
- 容器逃逸或危險預設掛載

## 回應原則

- 會盡量先確認問題是否可重現
- 修補完成前，會避免公開可直接利用的細節
- 若需要行為變更，會優先採保守預設值
